Scope

Scope will make your code stand up and reduce errors, so it is very important to grass this concept.

This is about security. Scoping parts of your code helps to improve efficiency, solves naming problems, track bugs and reduce them. We can define this as the lifetime and visibility of a variable.

In JavaScript, there are two types of scopes:

  • Global scope
  • Local Scope
GLOBAL scope

When you start writing in a document, you are already using the Global scope. If you define a variable outside of a function, this variable is in the global scope, and therefore accessible throughout the document.

const globalVariable = "some value";

Although you could easily write all your variables globally, it is advised not to. This is because there is a good change of naming collisions when using let or const>. if you are using var>, your second variable will overwrite the first one after being declared, so be careful!

LOCAL scope

These are variables defined and usable only in a specific part of your code. There are 2 types:

  • Function scope
  • Block scope
Function scope

When a variable is declared in a function, this can only be accessed within a function:

                        
                            function sayHi() {
                                const hi = "Hi reader!";
                                console.log(hi)
                            }
                            sayHi() // "Hi reader!"
                            console.log(hi) // error, hi is not defined
Block scope

When a let or const variable is declared within a brace { } you can access this only within that curly brace>

                    
                        {
                            const hi = "Hi reader!";
                            console.log(hi); // 'Hi reader!'
                        }
                        console.log(hi) // error, hi is not defined

To summarize, when a variable is declared in a function, this can only be accesses in the function. However, if you defined any inner function within another function, this inner function is called a closure. Feel free to find out more about closures on my post.
